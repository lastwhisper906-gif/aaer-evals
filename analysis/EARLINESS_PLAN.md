> Authored by Claude Code, pending human audit (GA-001 (b)). 2026-07-08, OWNER-GATE-E.
> **E2 첫 채점 전 커밋 (freeze-then-run).** 불변. 설계 모체: `docs/EARLINESS_DESIGN.md`.
> 범위 한정(§5-5): Claude 단일 파이프라인(claude-sonnet-5 핀).

## 0. 실무 질문

사기 신호는 폭로 직전에만 켜지는가, 아니면 여러 분기 전부터 자라는가? 같은 회사를
과거 여러 스냅샷에서 채점해 점수-시간 궤적 p(t)를 그린다 (t = 폭로까지 남은 분기).

## 1. 적격 케이스 (사전 고정)

- **wave-1 + wave-2 DETECTED 프라우드만** (플래그 발동 = 본채점 p≥50). 미탐 케이스는
  궤적의 기준점(폭로 시 플래그)이 없어 "선행시간" 정의가 성립 안 함 → 제외.
  - wave-1 detected: 본채점 p≥50인 실험군 (results_stats에서 확정).
  - wave-2 detected (7/9): WFT·CGI·MDXG·HAIN·OSIR·TNGO·UAA (CSC·BRX 미탐 제외).
- **MON 제외** (XBRL 의무화 직후 컷오프 — 뒤로 갈 데이터 없음, 1점 궤적, EARLINESS_DESIGN
  §2 실측 플래그).
- **스냅샷 최소 요건**: 잔존 제출 ≥6 및 10-K ≥1 (MON 본실행 수준 바닥).

## 2. 스냅샷 그리드 (제출 정렬, 결정론)

- 스냅샷 j의 컷오프 = 컷오프 이전 j번째 최신 XBRL 제출(10-K/10-Q) filed일 + 1일.
  (제출 정렬 이유: 이웃 점 차이 = 정확히 "제출 1건 제거" → 해석 가능.)
- **케이스당 스냅샷 상한 8** (더 깊어도 최근 8분기점 — 균형 패널). 스냅샷 0 =
  기존 본실행 점수 **재사용**(재호출 없음).
- 기존 `build_payload.load_pit_series` (filed ≤ cutoff 단일 필터) **무수정 재사용** —
  각 스냅샷은 그 시점 공개분만 → I2 스냅샷별 자동 성립. cutoff_guard 각 스냅샷 능동.
- **변형**: 교란(perturbed) 우선 (README 헤드라인 프레임 일치, 정체 암기 차단). 원본
  부록은 이 라운드 **미실행**(호출 절반). perturb_factor는 case_id 결정론 → 스냅샷 간
  동일 k(궤적 내 점 비교 가능). 케이스 ID: `case_NN_s{j}`.

## 3. 지표 (사전 지정)

- **1차: 탐지 선행시간** = 궤적이 플래그 임계 p≥50을 **처음 넘는 스냅샷**의 t
  (폭로까지 분기). "몇 분기 전에 신호가 켜지는가."
- **전체 궤적** p(t) 보고 (형태 판독). 집계: 폭로 전 8분기 OLS 기울기 · 실험군-대조군
  갭이 처음 10pp 넘는 시점 분포 · 조작 창 진입 전/후 평균 갭.
- **inside-noise 참조선**: RP-07 케이스별 σ(중위 6.3pp) — 이웃 점 차 이 밴드 안이면
  "변화 없음". k=1 8점 궤적은 **형태**만 읽고 점별 유의성 주장 안 함(잡음 밴드 병기).
- **d2 텍스트-증거 조기성 교차검증**: 이미 계산된 d2 earliness(1–4q, RP-10 Phase 3)와
  대조 — 재무 궤적 조기성이 텍스트 증거 조기성과 정합하는지.

## 4. 대조군 기준선

RP-01 확정 대조군 8 (점수·채점 동결)의 **동일 그리드 궤적** (교란만, 8×8=64) —
"대조군 궤적은 평평한가"가 실험군 상승의 대조. (대안: v2 확장군 일부 — 소유자 결정
사항이나 본 등록은 RP-01 8 채택, 재현성 우선.)

## 5. 호출 상한·절단 규칙 (사전 고정)

- 추정: 실험군 detected ~7–8사 × ≤8 스냅샷 ≈ 48–56 + 대조군 64 + 채점자 동수 ≈
  **112–160**. **전역 320 cap · E2 하위 cap 160**.
- **160 초과 투영 시 사전 절단 규칙**: **가장 깊은 이력부터 절단** — 케이스별 스냅샷을
  8→ 필요 시 6→4로 균일 축소(모든 케이스 동일 깊이 유지, 특정 케이스 편애 금지).
  대조군은 실험군 최종 깊이와 동일 깊이로 매칭. 이 규칙이 결과 독립을 보장.
- 채점 순서: **케이스 알파벳순**, 케이스 내 스냅샷은 **얕은 것(폭로 근접)부터**
  (절단이 깊은 쪽을 버리므로 얕은 점이 먼저 확정 = 선행시간 하한이 먼저 안정).

## 6. 결론 처리 (사전 명시)

- 상승 시작점이 **창 진입 전**이면: 조기 신호(실무 가치 큼). **마지막 1–2분기만**이면:
  폭로 근접 신호. **평평**하면: 신호 없음(정직 결과, 미화 금지).
- Karpoff et al. 집행 지연 맥락과 함께 프레이밍 (신호-폭로 시차의 구조적 배경).
  Issue #1 §earliness 신설. 성능 추정 아니라 **궤적 형태** 증거. **발행 안 함**.

## 7. 면책

단일 Claude 파이프라인, k=1 궤적(형태 판독). 대조군="비집행". 통제 아님.

### 주석 1 — §5 "실험군 detected ~7–8사"는 추정치, §1 규칙이 지배

§1의 적격 규칙(wave-1 + wave-2 DETECTED, 본채점 p≥50, MON 제외)의 기계 출력은
**13케이스**다. §5의 "~7–8사"는 detection 확정 전에 쓰인 추정으로, 자신이
추정하는 규칙에 의해 대체된다(superseded). 구속 로스터 (cutoff 병기):

| # | wave-1 detected 6 | cutoff | # | wave-2 detected 7 | cutoff |
|---|---|---|---|---|---|
| T11 | OFIX | 2013-07-28 | T04 | WFT | 2011-02-28 |
| T13 | HTZ | 2014-05-12 | T19 | OSIR | 2015-11-05 |
| T16 | ICON | 2015-08-09 | T22 | TNGO | 2016-03-06 |
| T17 | MRVL | 2015-09-10 | T23 | HAIN | 2016-08-14 |
| T21 | SCOR | 2016-02-28 | T26 | MDXG | 2016-12-14 |
| T28 | KHC | 2019-02-20 | T24 | CGI | 2017-04-04 |
| | | | T29 | UAA | 2019-11-02 |

로스터 소속 ≠ 스냅샷 가용: §1 스냅샷 최소 요건(잔존 ≥6·10-K ≥1)의 기계적
귀결로 WFT는 스냅샷 0 (XBRL 4건 — MON과 동일한 의무화 경계 사유),
실측 그리드 = 실험군 91 + 대조군 55 (docs/E2_PREFLIGHT.md §4).

### 주석 2 — §5 "채점자 동수"는 E2에 비작동(inoperative)

E2의 사전 지정 지표(§3: p≥50 최초 돌파 시점·궤적 형태·OLS 기울기·갭 분포)는
**피평가자 원점수만 소비**한다 — 채점자 차원(R/H 등)은 어떤 지표에도 등장하지
않는다 (D49의 "점수 통계는 채점 0" 논리와 동일). DESIGN에서 상속된 "채점자
동수" 문구는 E2에 비작동이며, **기록 예산 = 146 evaluatee + 0 grader**
(동수 해석 시 292 > 하위 cap 160 — 문구 자체가 §5의 cap과 모순).
이 주석은 지출을 **좁히기만** 하고 지표·임계·판정 규칙을 바꾸지 않는다 —
커버리지 노트와 동급의 수리이며, 결과가 존재하기 전에 이뤄진다.
