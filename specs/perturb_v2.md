## 1. 동기 — v1의 남는 지문

상수배 재스케일(v1, `perturb_factor`)은 모든 비율·성장률을 정확히 보존하고
**날짜를 건드리지 않는다**. 제출물 연대기의 절대 날짜 시퀀스는 준-유일한
회사 지문이다 (잔여 name-ID의 기제 후보). v2는 절대-날짜 매칭을 파괴하되
**모든 간격 기반 신호**(지연 제출 갭·NT 타이밍·제출 리듬)를 보존하는
날짜 이동 층을 추가한다.

## 2. 오프셋 규칙 (결정론, HIPAA date-shift 스타일)

케이스별 `offset_days = sign × 7 × w`:

- `h = sha256((case_id + "dateshift-v2").encode()).digest()` —
  기존 `perturb_factor`(D8)와 동일한 스타일의 케이스별 결정론 유도.
- `w = 26 + (int.from_bytes(h[:4], "big") % 53)` → w ∈ [26, 78] 정수
  (오프셋 크기 182–546일, **주 단위 전체**).
- `sign = +1 if h[4] & 1 else −1`.
- **주 단위 이동은 요일을 보존한다** — 제출물은 평일에 몰리며, 토요일
  10-K는 그 자체가 탐지 가능한 인공물이다.

## 3. 균일성 규칙 (전 날짜 단일 오프셋)

피평가자가 보는 **모든** 날짜에 동일 오프셋 적용:

- XBRL 기간 `start`/`end`/`filed` · 연대기 `filing_date` · 과제문의
  `cutoff_date` (case 필드).
- 따라서 모든 컷오프 비교와 간격은 **정확히 불변**이다 (전 날짜에 같은
  상수를 더하므로 순서·차이 보존은 항등식).
- **순서 명시**: 실제 cutoff guard는 상류(진짜 날짜)에서 계속 작동한다.
  이동은 가드 검사 **후**, 페이로드 렌더 시점에서만 일어난다
  (`build_payload(perturb=True)` → `shift_payload(...)`).
- **accession 마스킹 (사전 등록 추가)**: accession 번호는 제출 연도를
  내장한다 (`0001193125-15-…`) — 이동된 filed와 모순되고 진짜 연도를
  누출한다. v2 렌더에서 accession은 케이스 내 순차 중립 ID(`acc-001`…)로
  치환한다.

## 4. 잔여 인공물 (정직 절 — 수용한 트레이드오프)

1. **회계연도 말이 비표준 날짜에 떨어진다** (1월 17일 연말 등) — 경미한
   비개연성 신호. 수용.
2. **거시 맥락 불일치** — 2008년형 침체 궤적이 더 이상 2008년과 정렬되지
   않는다. 수용.
3. 고려-기각 대안: **월 단위 이동**(요일 파괴 + 월말/분기말 경계 인공물),
   **태그별 노이즈**(회계 항등식 파괴 — 자산≠부채+자본이 되는 순간 과제
   자체가 무효). 둘 다 기각.

## 5. 효과성 지표 (사전 등록 — 미래 미터링 엔드포인트)

- 엔드포인트: 기존 name-ID 프로브를 v2 페이로드로 재실행 (동일 프로브
  문구, 동일 k) → **v2 name-ID rate vs 동결 v1 rate (wave-1 50% [15/30] ·
  wave-2 21.9% [7/32, 동결 name_match 규칙])** 비교.
- **호출 산술 (repo 계수 — 추정 아님)**: wave-1 프레임 30
  (`name_probe_results.json` n_probes=30 = 8 treatment + 22 control) +
  wave-2 32 (`cases_wave2.json` 전건) = **62 호출**. 홀드아웃은 v1 name-ID
  기준선이 없어(정체 가시 프레임) 비교군에서 제외.
- **지금 실행 가능한 무비용 위생 검사**: 원천 JSON의 진짜 날짜 문자열이
  v2 렌더 페이로드에 하나도 생존하지 않음을 문자열 스캔으로 단정
  (`test_date_shift.py` no-leak 테스트 — 원본 accession 부재 포함).

## 6. 오염 프로브 주석 (범위 밖 등록)

guided-completion / 시간-분할 프로빙은 별도의 미래 미터링 진단으로
등록한다 — logprob 기반 min-k% 계열은 배포 API에서 불가하므로 텍스트
레벨 변형만 가능. 이 세션 실행 범위 밖.

## 7. 테스트 계약 (`pipeline/test_date_shift.py`)

1. 결정론 (동일 case_id → 동일 오프셋, 스냅샷 고정).
2. 주 단위 속성 (`offset % 7 == 0`, 182 ≤ |offset| ≤ 546) + 표본 케이스
   집합에서 양·음 부호 모두 출현.
3. 간격 보존 (이동 전후 모든 날짜쌍 차 동일) + 요일 보존.
4. 컷오프 비교 불변 (`filed <= cutoff` 관계 전건 동일).
5. no-true-date-leak 문자열 스캔 (원본 ISO 날짜·원본 accession 전부 부재).
