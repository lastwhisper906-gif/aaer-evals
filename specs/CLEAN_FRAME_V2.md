# CLEAN_FRAME_V2.md — 완전 탈익명화 프레임 v2 (사전 등록 — 실행 보류, 기본 HOLD)

> **지위: 사전 등록 스펙. 실행되지 않았고, 실행 승인도 아니다 — 기본 상태는
> HOLD.** 발사는 자체 FREEZE_REV + 자체 spend gate(소유자 서명) 전용 (§7).
> 외부 검토 약점 1·2 대응 — v1 부분 탈익명화(D99,
> `docs/V1_PARTIAL_DEIDENTIFICATION_NOTE.md`)의 6개 지적 전부를 설계로 반영.

## 0. 정직한 위치 규정

**전향 봉인 트랙 레코드(첫 seal 2026-11-15)는 암기 문제를 구성적으로
해소한다** — 봉인 시점에 결과가 존재하지 않으므로. 본 스펙이 구제하는
것은 그것이 아니라 **회고 wave들(TASK 1)의 측정 품질**이다: v1 교란
프레임의 잔여 정체 채널을 닫아, "재무 패턴만으로의 분리"를 지금보다
덜 오염된 프레임에서 재측정할 수 있게 한다. v1 동결 산출물·발행 수치는
무접촉 — v2는 병행 측정이지 대체가 아니다.

## 1. 합성 accession 치환 (검토 요소 ①)

- v1 잔존 채널: 원본 accession(`{filer-CIK10}-{YY}-{seq}`)이 제출자 CIK와
  제출 연도를 인코딩 — 문서 ID 하나가 정체 전체를 복원 (D99).
- v2 규칙: 전 accession을 **합성 ID로 치환**한다. 형식
  `9{XXXXXXXXX}-{RR}-{NNNNNN}`: 선두 `9` + 케이스별 시드 고정 난수 9자리
  (실존 CIK 공간 상위 블록과 비충돌 — SEC CIK는 현행 10자리 좌측 0 패딩,
  선두 9 배정 없음 실측 후 발사 전 재확인을 게이트에 포함), `RR` = §2의
  상대 연차(절대 연도 아님), `NNNNNN` = 케이스 내 문서 일련.
- **충돌 안전**: 케이스 내·케이스 간 전단사 매핑, 실존 accession과의 충돌
  0을 발사 전 위생 스캔으로 기계 확인 (Q-F05 위생 스캔 선례). 매핑 테이블은
  scoring/ 쪽 비밀로 보관 — 피평가자 표면 반입 금지 (cli_client 마커 가드에
  원본 accession 접두 패턴 추가).
- 대안 경로(등록만): accession **전면 제거**. 문서 참조 무결성이 깨져 인용
  검증이 약화되므로 기본안은 치환, 제거는 fallback.

## 2. 상대 회계 시간축 (검토 요소 ②③)

- v1 잔존 채널: 실제 제출 날짜·간격의 연대기가 회사 고유 지문 (D99: 연대기
  732건 유지 실측).
- v2 규칙: 전 날짜를 **상대 회계 축**으로 사상 — 기준점 t0 = 케이스 컷오프
  직전 회계연도말, 모든 날짜는 `FY-k Qq` + t0로부터의 주 단위 오프셋으로
  표기. 절대 연도·월·일 전면 제거.
- **회사 고유 회계 연대기 제거 (요소 ③)**: 회계연도말 월(1월말 결산 등
  식별자), 제출 요일 패턴, 불규칙 간격(정정 클러스터의 실제 날짜)은 상대
  축에서 **분기 격자로 스냅**한다 — 보존되는 것은 순서·분기 정렬·이벤트
  간격의 분기 수뿐. 스냅 규칙은 결정론(시드 무관).

## 3. 구조 보존 다축 변환 (검토 요소 ④)

- v1 잔존 채널: 상수 스케일링은 비율·추세를 완전 보존 — 규모 자체가 지문.
- v2 규칙: 케이스별 시드 고정으로 **축별 상이 변환**을 적용하되, 아래
  **불변량(invariants)은 보존을 기계 검증**한다:
  - 보존 (판단에 필요한 구조): 동일 축 내 시계열 추세 방향·상대 변화율,
    축 간 비율의 시계열(매출채권/매출, 재고/매출원가, CFO/NI 등 스크리닝
    비율 전부), 부호, 분기 계절성 패턴.
  - 파괴 (정체 지문): 절대 금액 수준(축별 상이 계수 — 상수 아님), 자릿수
    지문(선행 자리 분포), 보고 단위 습관(천/백만 단위 반올림 패턴).
  - 변환족 후보: 축 그룹별 로그-공간 아핀 변환 (비율 그룹 단위로 동일
    계수 — 그룹 내 비율 보존, 그룹 간 절대 관계 파괴). 정확한 변환족·그룹
    분할은 발사 전 FREEZE_REV에서 고정 (본 등록은 불변량 계약까지).
- 검증 테스트(발사 전 커밋): 변환 전후 스크리닝 비율 시계열의 동일성
  (허용 오차 float 정밀도), 절대값 비복원성(계수 비공개 하에서).

## 4. 정보량·재식별률 측정 (검토 요소 ⑤)

- **사전/사후 정보량**: 변환 전후 페이로드에 대해 결정론 기준선(Beneish M
  성분·Dechow F 성분·스크리닝 비율)의 순위 보존율을 계산 — 판단에 쓰이는
  정보가 변환에서 살아남는가의 기계 척도.
- **재식별률**: v1 name-ID 프로브와 동일 문구·동일 k의 프로브를 v2
  프레임에 실행, v1 대비 (wave-1 50% [15/30] · wave-2 21.9% [7/32] 기준선
  — Q-F05 등록 엔드포인트와 동일 규약).
- **수용 임계는 본 등록에서 숫자를 박지 않는다** — 발사 FREEZE_REV에서
  결과 열람 전 고정한다 (사전 고정 원칙; 지금 박으면 v2 설계 반복 중
  임계가 결과를 본 뒤 조정될 위험이 오히려 커진다). 등록되는 것은 임계의
  **형태**: 재식별률 상한 (v1 대비 감소 필수 + 절대 상한) AND 정보량
  보존율 하한의 이중 조건.

## 5. 비유도 인지 프로브 (검토 요소 ⑥)

- 금지: "이 회사의 이름을 추측하라" 류 유도 문구 — 유도 자체가 재식별
  탐색을 훈련시키는 프레임.
- v2 프로브: 개방형 비유도 형식("이 재무 패턴에 대해 알고 있는 바를
  기술하라" 계열 — 정확 문구는 발사 FREEZE_REV에서 동결), 판정은 응답
  내 실명·실사건 자발 출현의 동결 규칙 매칭. name_match 동결 규칙 승계
  (rename 처리는 Q-E02 각주 규약 유지).

## 6. 산출물·비교 계약

- v2 프레임 재채점은 wave-1·wave-2 실험군+대조군 전건 (동일 프로토콜 —
  대조군 필수 불변식). 엔드포인트: v2 분리 통계 vs v1 교란 프레임 동결값
  (병기 — v1 무수정), v2 재식별률 vs v1.
- 어느 방향의 결과든 발행 표면 반영은 소유자 서명 diff 전용.

## 7. 스코프 가드·예산 (등록만 — 발사 아님)

- **신규 실험 = 자체 FREEZE_REV + 자체 spend gate** (D100/D106 선례).
  본 스펙 커밋은 어떤 호출도 승인하지 않는다.
- 예산 형태(등록): 재채점 (30+32)×2프레임 아님 — v2 단일 프레임 62 +
  프로브 62 ≈ **124 호출** 차수. 정확 산술은 발사 FREEZE_REV에서 케이스
  로스터 기준 재계수. zero-metered 명령(D102) 하 구독 OAuth 전용.
- 케이스 확장 없음(8/9 유지), 파이프라인 재설계 아님(변환은 build_payload
  perturb 경로의 교체 1지점) — 스코프 가드 §8 내.
