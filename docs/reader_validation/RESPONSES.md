- 집계 갱신 규율: ③~⑤가 서명되지 않은 행은 집계에 넣지 않는다 (전사만
  된 응답은 "응답률" 행에만 계수).
- 판정 근거는 반드시 verbatim 인용을 동반한다 — 인용 없는 판정은 서명
  대상이 아니다 (PROJECT.md §7 서명 규칙).
- 오버라이드(세션 전사 오류 정정 포함)는 `scoring/overrides.md`에 기록.

## 3. 마감 규율

- 응답 마감 = 발송일 + 10일 (TARGET_LIST 규약). 마감 후 무응답도 데이터다 —
  "응답 0"은 실패가 아니라 Tier 3의 실측이며 그대로 발행 표면에 기록한다
  (나쁜 결과 미화 금지 — PROJECT.md §10).
- 본 결과는 Claude 기반 단일 파이프라인에 한정 (§5-5). 채점: Claude 보조 +
  인간 최종 확정.
